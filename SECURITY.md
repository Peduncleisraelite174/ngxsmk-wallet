# Security Policy

We take the security of NGXSMK Wallet seriously. This document outlines our policy regarding security vulnerabilities, how to report them, and what to expect in response.

## Supported Versions

We actively support and provide security updates for the following versions of NGXSMK Wallet:

| Version | Supported |
| ------- | --------- |
| v1.x    | Yes       |
| < v1.0  | No        |

## Reporting a Vulnerability

If you discover a security vulnerability in NGXSMK Wallet, please report it privately. **Do not open a public GitHub issue for security vulnerabilities.**

Please email your report to **[ngxsmk@gmail.com](mailto:ngxsmk@gmail.com)**.

To help us triage and resolve the issue quickly, please include:
- A descriptive title and summary of the vulnerability.
- Step-by-step instructions (or a proof-of-concept script/exploit) to reproduce the issue.
- The platform(s) affected (Windows, macOS, Linux).
- The version of NGXSMK Wallet you used.
- Any potential impact or implications of the vulnerability.

## Our Response Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
2. **Investigation**: We will investigate the issue and determine if it is indeed a vulnerability, estimating its severity.
3. **Fix & Release**: If verified, we will work on a fix. We aim to release a patched version within 30 days of report verification.
4. **Disclosure**: Once a patch is released, we will coordinate public disclosure of the vulnerability with you.

We appreciate your help in keeping NGXSMK Wallet secure for everyone!
