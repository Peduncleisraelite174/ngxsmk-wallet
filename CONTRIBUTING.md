# Contributing to NGXSMK Wallet

Thank you for your interest in contributing to NGXSMK Wallet! Contributions from the community help make this privacy-first manager even better.

Please note that our codebase is structured into two parts:
1. **Frontend (React / TypeScript)**: Open-source under the MIT License, located in the `ngxsmk-wallet-project` repository.
2. **Backend (Rust)**: Proprietary (All Rights Reserved) and licensed for pre-built binaries.

Therefore, codebase contributions are primarily accepted for frontend enhancements, security scanning, styling, translation, documentation, and bug reporting.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md) in all interactions.

## How to Contribute

### 1. Reporting Bugs
- Search existing issues to ensure the bug hasn't been reported yet.
- Open a new issue with a clear title and description. Include steps to reproduce, expected behavior, and screenshots if applicable.

### 2. Suggesting Enhancements
- Check if your feature idea is already in the roadmap or discussed in issues.
- Describe the feature, the use case, and why it would benefit all users.

### 3. Submitting Code Changes (Frontend)
- Fork the `ngxsmk-wallet-project` repository.
- Create a new branch named after your feature or fix (e.g., `feature/custom-icons` or `fix/clipboard-timeout`).
- Follow the existing project code style (Prettier, ESLint).
- Ensure your changes do not break existing tests.
- Submit a Pull Request targeting the `main` branch.

### 4. Improving Documentation
- Typos, grammar corrections, clarifications, and improved installation instructions are welcome.
- Edit files directly and submit a PR.

## Development Setup

To work on the frontend code:
1. Clone the frontend project repository.
2. Run `npm install` to install dependencies.
3. Start the Vite dev server with `npm run dev`.
4. Run tests with `npm run test` or `npm run test:e2e`.

Thank you for helping keep digital identity secure and private!
